# Frameworks Narratifs pour Data Stories

## 1. Le Pyramid Principle (Barbara Minto)

### Structure SCQA

La structure la plus utilisée en consulting (McKinsey, BCG) :

```
┌─────────────────────────────────────┐
│           SITUATION                 │
│     (Contexte partagé)              │
├─────────────────────────────────────┤
│          COMPLICATION               │
│     (Problème ou changement)        │
├─────────────────────────────────────┤
│           QUESTION                  │
│     (Question clé soulevée)         │
├─────────────────────────────────────┤
│            ANSWER                   │
│     (Réponse/Recommandation)        │
└─────────────────────────────────────┘
```

**Exemple** :
- **Situation** : "Notre entreprise est leader sur le marché français depuis 10 ans"
- **Complication** : "Un nouveau concurrent agressif a capturé 20% du marché en 18 mois"
- **Question** : "Comment devons-nous répondre pour protéger notre position ?"
- **Answer** : "Nous recommandons une stratégie en 3 axes : innovation produit, fidélisation client, expansion géographique"

### La Pyramide des Arguments

```
                 MESSAGE PRINCIPAL
                      /    \
                     /      \
              ARGUMENT 1   ARGUMENT 2   ARGUMENT 3
               /     \       /    \       /     \
           Data    Data   Data   Data   Data   Data
```

**Règles** :
1. Commencer par la conclusion (Answer First)
2. Grouper les arguments par thème (MECE - Mutually Exclusive, Collectively Exhaustive)
3. Chaque niveau soutient le niveau supérieur
4. Maximum 3-5 points par niveau

## 2. Le Hero's Journey Adapté aux Données

### Les 8 Étapes

```
1. MONDE ORDINAIRE
   └── État actuel, baseline

2. APPEL À L'AVENTURE
   └── Problème identifié, opportunité

3. REFUS DE L'APPEL
   └── Obstacles, résistances prévisibles

4. RENCONTRE DU MENTOR
   └── Les données comme guide

5. PASSAGE DU SEUIL
   └── Décision d'agir, point de non-retour

6. ÉPREUVES ET ALLIÉS
   └── Challenges à surmonter, ressources nécessaires

7. ÉPREUVE SUPRÊME
   └── Le défi majeur, le risque

8. RETOUR AVEC L'ÉLIXIR
   └── Bénéfices, transformation accomplie
```

**Application** :
- Le "héros" peut être l'entreprise, le client, ou le produit
- Les données sont le "mentor" qui révèle le chemin
- La transformation est le résultat mesurable

## 3. Le Framework Problem-Solution-Benefit

### Structure Simple et Efficace

```
┌─────────────────────┐
│      PROBLEM        │ ← Douleur actuelle (chiffres)
├─────────────────────┤
│      SOLUTION       │ ← Ce que nous proposons
├─────────────────────┤
│      BENEFIT        │ ← Résultats attendus (chiffres)
└─────────────────────┘
```

**Variante étendue : PASTOR**
- **P**roblem : Identifier le problème
- **A**mplify : Montrer les conséquences de l'inaction
- **S**tory : Raconter un cas concret
- **T**estimony : Preuves et témoignages
- **O**ffer : Présenter la solution
- **R**esponse : Appel à l'action

## 4. Le Before-After-Bridge (BAB)

### Pour les Transformations

```
BEFORE (Avant)
├── État actuel problématique
├── Données qui illustrent le problème
└── Impact négatif quantifié

AFTER (Après)
├── Vision de l'état désiré
├── Bénéfices quantifiés
└── Témoignages ou benchmarks

BRIDGE (Pont)
├── Comment passer de Before à After
├── Les étapes concrètes
└── Le plan d'action
```

## 5. Le Framework des Contrastes (Duarte)

### Alternance Tension-Résolution

```
    │
    │    ╱╲      ╱╲      ╱╲
    │   ╱  ╲    ╱  ╲    ╱  ╲    ╱
    │  ╱    ╲  ╱    ╲  ╱    ╲  ╱
    │ ╱      ╲╱      ╲╱      ╲╱
    │╱
    └────────────────────────────►

    Ce qui est  →  Ce qui pourrait être
    (What is)      (What could be)
```

**Transitions suggérées** :
- "Aujourd'hui nous sommes ici... mais imaginons..."
- "Les données montrent X... cependant Y est possible..."
- "Le risque est réel... mais l'opportunité l'est aussi..."

## 6. Le Framework Analytique à Synthétique

### Du Détail au Message

```
ANALYSE (Bottom-up)           SYNTHÈSE (Top-down)

Données brutes                Message principal
    ↓                              ↓
Insights                      Arguments clés
    ↓                              ↓
Patterns                      Preuves
    ↓                              ↓
Conclusions                   Données support
```

**Conseil** : Analyser bottom-up, présenter top-down

## 7. Le Modèle STAR pour les Cas Concrets

### Structurer les Exemples

- **S**ituation : Contexte spécifique
- **T**ask : Défi à relever
- **A**ction : Ce qui a été fait
- **R**esult : Résultats mesurables

**Exemple** :
- **Situation** : "Client retail avec 15% de churn annuel"
- **Task** : "Réduire le churn de 5 points"
- **Action** : "Programme de fidélité basé sur données comportementales"
- **Result** : "Churn réduit à 8%, ROI de 340%"

## 8. Structures Temporelles

### Chronologique
```
Passé → Présent → Futur
```
Utile pour montrer une évolution

### Rétrospective
```
Résultat → Comment nous y sommes arrivés
```
Utile quand le résultat est impressionnant

### Parallèle
```
Scénario A ║ Scénario B
    ↓      ║     ↓
Résultat A ║ Résultat B
```
Utile pour comparer des options

## Choisir le Bon Framework

| Contexte | Framework Recommandé |
|----------|---------------------|
| Recommandation stratégique | Pyramid Principle (SCQA) |
| Pitch commercial | Problem-Solution-Benefit |
| Transformation digitale | Before-After-Bridge |
| Keynote inspirationnelle | Hero's Journey |
| Comité de direction | SCQA + Contrastes |
| Rapport mensuel | Chronologique + STAR |
